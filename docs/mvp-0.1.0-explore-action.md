# 0.1.0 探索动作说明

## 背景

Spotlight 不能假设“打开的项目目录一定是代码仓库”。

真实场景里，项目目录可能是：

- 空目录
- 只有 Word、Excel、PPT、PDF 等文档
- 只有零散图片、压缩包或交付资料
- 已有源码，但结构不完整

因此，在项目打开后，需要提供一个明确的“探索”动作，让 Agent 先理解当前路径，再给出建议任务列表，而不是直接假设要写代码。

## 0.1.0 的最小落地

在项目视图中增加 `探索目录` 按钮。

点击后，系统会为当前项目创建一条预制任务：

- 标题：`探索当前目录并生成建议任务`
- 状态：`open`
- 归属：当前项目

该任务的预制提示词要求 Agent 输出：

1. 当前目录内容摘要
2. 可识别的技术栈、文档类型或交付物
3. 主要风险、信息缺口和需要人工确认的地方
4. 建议的任务列表，按优先级排序，标题和说明使用中文

## 提示词约束

预制提示词必须明确提醒 Agent：

- 不要假设当前目录一定有源码仓库
- 目录可能为空，也可能只有 Office、PDF、图片等资料
- 不可直接读取的文件，不允许臆造内容
- 可以基于文件名、目录结构、周边文本和元数据做谨慎判断
- 如果目录几乎为空，要如实说明，并给出下一步建议

## 后续扩展

后续版本可以在“探索”基础上继续扩展：

- 一键把建议任务列表落成正式任务
- 按目录或 workspace root 分区探索
- 生成结构化探索摘要，沉淀到项目档案
- 结合不同大模型 Provider 做低成本摘要和高质量分析的分层路由
