# Bootstrap Notes

This implementation intentionally starts small.

## First Working Loop

1. seed tasks from the design documents
2. display tasks on a board
3. claim a task to an agent
4. invoke an agent stub
5. update task status and activity log

## Why This Shape

The team can use this smallest loop to coordinate the real build-out of Spotlight itself.

That lets the product start behaving like the thing it is trying to become.
